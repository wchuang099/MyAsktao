# 梦幻西游脚本开发笔记：自动换号与任务链恢复逻辑

## 一、 课程核心目标 (Core Objectives)

本课主要解决在多账号自动化执行过程中，由于**脚本重启、线程异常**或**自动换号**导致的任务进度丢失问题。

1. **动态任务链配置**：用户可自由组合打图、押镖、五行等任务顺序。
2. **断点续传逻辑**：脚本意外停止后再启动，能自动识别已完成任务，从当前进度继续。
3. **跨账号配置复原**：实现“一次设置，全号通用”，换号后自动重置任务链缓存。

------

## 二、 任务链 UI 交互设计 (UI Design)

### 1. 双列表框架构

- **全部任务列表 (`list_all`)**：展示脚本支持的所有功能模块（如打图、押镖等）。
- **执行流程列表 (`list_flow`)**：展示用户自定义的执行顺序。
  - **交互逻辑**：双击 `list_all` 项目，自动插入数据至 `list_flow`；右键点击 `list_flow` 项目执行删除。
  - **显示设置**：必须将超级列表框类型设置为 **报表模式 (Report Mode)** 并在“整行选择”属性中设为 `True`。

### 2. 任务链序列化 (Serialization)

在 UI 窗口销毁（关闭）时，需遍历 `list_flow`，将任务名称通过特定分隔符（如 `|` 或 `-`）拼接成文本字符串，并写入本地 INI 配置。

------

## 三、 任务执行与恢复算法 (Execution & Recovery Algorithm)

### 1. 任务缓存机制

为了防止脚本重启后重复执行已完成的任务，引入“**动态剔除**”逻辑：

- **读取配置**：主引擎启动后，从配置读取原始任务链。
- **任务剔除**：每当一个子任务（如打图）执行完毕，立即从配置文件的任务链文本中删除该任务标识，并重新写入。

## 四、 跨账号“自动复原”逻辑 (Cross-account Recovery)

### 1. 缓存与镜像分离

为了在换号后能够重新开始全套任务，必须建立两个配置项：

- **`Task_Flow` (实时流)**：随着任务完成而不断缩减，用于断点恢复。
- **`Task_Mirror` (原始镜像)**：存储用户最初设定的完整任务链。

### 2. 换号重置触发

当 `Task_Flow` 变为空字符串且执行自动换号时：

1. **检测空链**：识别到当前账号任务已全部结束。
2. **镜像复位**：将 `Task_Mirror` 的内容重新拷贝给 `Task_Flow`。
3. **新号启动**：下一个账号登录后，读取的是被重置后的完整任务链。

------

## 五、 关键避坑指南 (Troubleshooting)

1. **索引偏移问题**：易语言计次循环索引从 `1` 开始，而超级列表框索引从 `0` 开始。在遍历删除时务必注意 `Index - 1` 的对应关系。
2. **UI 线程安全**：在多线程脚本中，直接操作 UI 控件极易崩溃。**强烈建议使用标签反馈或消息通知机制**进行 UI 更新。
3. **配置写入时机**：不要仅在任务全部结束时写配置。应在“每一个子任务”结束时立即同步本地文件，以应对随时可能发生的掉线或崩溃。
4. **路径纠错**：使用 `取运行目录()` 获取相对路径。避免在系统重装或目录移动后，因路径失效导致配置无法读取。

------
