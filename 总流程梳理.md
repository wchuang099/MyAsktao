## 二、核心模块实现逻辑

### 2.1 多线程架构设计

**易语言模式迁移**:

```
全局线程信息数组 → Python类实例列表
- 每个线程对应一个游戏窗口
- 独立的大漠对象
- 独立的配置节点
- 独立的鼠标漂移修正参数
```

**线程生命周期**:

1. **初始化**: 创建大漠对象 → 加载字库 → 绑定窗口
2. **运行**: 状态机循环(读取任务→寻路→战斗→完成)
3. **暂停/恢复**: 通过标志位控制
4. **停止**: 释放资源

### 2.2 鼠标漂移解决方案(核心技术)

**问题分析**: 后台绑定VNC窗口时,绝对坐标移动存在系统级偏差

**解决思路**:

```
原始移动: MoveTo(x, y) → 实际落点存在随机漂移

修正方案:
1. 初次移动: 绝对移动到目标点附近
2. 位置校准: 
   - 字库识别法: 识别鼠标光标特征字
   - 图像匹配法: 模板匹配鼠标指针图片
3. 误差计算: Δx = 目标x - 实际x, Δy = 目标y - 实际y
4. 循环修正: 相对移动(Δx, Δy) 直到误差<阈值
5. 自学习: 记录不同区域的修正值,下次直接预补偿
```

**关键点**:

- 鼠标在不同屏幕区域(左/中/右,上/中/下)漂移规律不同
- 需要建立**分区校正模型**
- 新区卡顿和老区流畅需要自适应调整修正策略

### 2.3 图像识别系统

**三层识别机制**:

```
第1层: 快速找图
- 大漠找图
- 用于UI元素检测(按钮/对话框)
- 特点:速度快,精度一般

第2层: 精确找图  
- OpenCV模板匹配
- 用于NPC/物品识别
- 特点:可调阈值,支持多图匹配

第3层: OCR文字识别
- 字库查找(快但需要预建字库)
- PaddleOCR/Tesseract(慢但通用)
- 用于任务内容/坐标/地图名解析
```

**识别容错策略**:

- 多图备选: "图片A|图片B|图片C" 匹配任一即成功
- 区域限定: 动态计算识别区域,减少误判
- 超时重试: 识别失败→延时→重试→超时降级处理

### 2.4 地图导航系统

**核心逻辑**:

**code**

复制

插入

新建文件

保存

应用



```
1. 获取当前位置
   - OCR识别左上角"地图名[x,y]"
   - 或内存读取坐标数据

2. 路径规划
   - 判断是否跨地图 → 调用传送(驿站/飞行旗)
   - 同地图 → 计算小地图点击点

3. 坐标换算
   - 游戏坐标 → 屏幕像素坐标
   - 公式: screen_x = 小图_x + (游戏_x / 地图最大_x * 小图宽度)
          screen_y = 小图_y - (游戏_y / 地图最大_y * 小图高度)

4. 寻路等待
   - 持续检测位置是否变化
   - 连续N次坐标不变 → 判定到达
```

**跨地图处理**:

- 预定义地图节点配置(传送点坐标、地图连接关系)
- 飞行旗/驿站优先级策略
- 未知地图的兜底方案(步行/问路)

## 三、关键难点与解决方案

### 3.1 配置隔离与多开

**问题**: 多个游戏窗口需要独立配置,避免参数冲突

**方案**:

- 配置文件结构: `config.ini` 按"窗口_1/窗口_2"分节点
- 运行时通过窗口索引动态读取对应节点
- 全局配置(如分辨率) + 窗口配置(如VNC名称、鼠标修正值)

### 3.2 对话框自动化

**难点**: 游戏对话框内容动态,按钮位置不固定

**方案**:

1. **界面判定**: 检测对话框标识图片(如"关闭按钮")
2. **按钮识别**: OCR识别按钮文字("接受/拒绝/取消")
3. **相对点击**: 基于对话框位置计算按钮相对坐标
4. **超时处理**: 对话框未弹出→点击NPC→重试→超时放弃

### 3.3 NPC遮挡处理

**场景**: NPC被其他玩家或物体遮挡,无法正常点击

**处理策略**:

1. **屏蔽功能**: Alt+H 屏蔽其他玩家
2. **视角拉近**: 点击NPC附近位置,拉近摄像机
3. **Ctrl机制**: Ctrl+点击NPC,显示所有重叠NPC列表,OCR识别目标NPC名并点击
4. **多角度尝试**: 从不同方位接近并点击

### 3.4 新老区性能适配

**问题**: 新区画面卡顿,老区流畅运行

**自适应策略**:

- **检测机制**: 统计位置变化频率,卡顿检测(连续N次位置不变)

- 新区优化

  :

  - 增大延时参数
  - 增大识别容错阈值
  - 减少高频循环
  - 跳过部分校准步骤

- **老区保持**: 使用默认的高性能参数

## 六、总结与建议

**核心设计理念**:

1. **模块化**: 将大漠插件的各个功能封装为独立Python模块
2. **状态机**: 复杂流程使用状态机管理,避免嵌套if-else
3. **容错性**: 每个关键步骤都应有重试机制和超时处理
4. **可配置**: 参数外置,支持不同游戏/区域/性能需求

**易语言→Python迁移要点**:

- 易语言"子程序" → Python函数/方法
- 易语言"自定义数据类型" → Python类
- 易语言"全局数组" → Python列表/字典
- 易语言"判断循环" → Python while循环

**最大挑战**: 鼠标漂移问题需要重点攻克,建议先做原型验证

这套架构可以直接应用到其他游戏脚本开发,只需调整具体任务逻辑和图像识别模板即可。

    # 以下是各个状态对应的辅助函数(简化版)

2. 状态切换机制

  读取当前任务 → (有任务) → 选择传送方式 → (跨地图) → 跨地图导航
     ↓                      ↓                    ↓
   (无任务)              (同地图)            同地图寻路
     ↓                      ↓                    ↓
    接取任务              同地图寻路         点击强盗NPC
     ↓                                          ↓
    接取成功                                    战斗循环
     ↓                                          ↓
    返回读取任务                              任务完成
     ↓                                          ↓
    返回读取任务 ←──────────────────────────────┘
